CSE104 Data Structure

Chenxu Fu 1509284

Summary

####Designing

  • The program utilized HashMap as its major data structure. Considering the system needs to store ‘product’, ‘vendor’,’price’ while HashMap can only store binary-element information, a nested HashMap acts as the outer HashMap’s value.
  • In all the manipulation methods used, it takes user input to search the collection. Since product is the key, it can quickly determine whether the collection contains products of inputted name. Then similarly, by matching the key of nested HashMap, it can locate the product with designated vendor if it exists.
  • For Add() method, there are 3 scenarios. If there is already the same product by same vendor, system rejects the entry. If the same product exists but by no same vendor, retrieve the nested HashMap and add the new vendor/price pair to it. If it is a new item, create a new HashMap and add the vendor/price pair to it, and nest the HashMap to the outer ItemHashMap as its value, product name as its key.
  • For Ask() and Delete() methods, differentiate two scenarios that vendor is inputted and not. If vendor is input, get or modify the designated product if available. If vendor is not inputted, traverse the collection and get all products by inputted product.
  • For DisplayAll() method, it utilized Set to traverse the collection. By assigning the keySet of both the inner and outer HashMap to a new Set. For-Each method can be used to iterate through the collection and to print the information.
  • Hashmap put() and get() operation time complexity is O(1) with assumption that key-value pairs are well distributed across the buckets. So in comparison, it is more efficient when removing entries than ArrayList on remove() and contains() – O(n). Set is also O(1) if using HashSet.
    ####Testing
  • To test the program, I added a few lines of code to preset a product as it initialized and tested a variety of scenarios as follows:
    1. Basic formatting issues including wrong commands, invalid prices.
    2. Try Add() to add an identical product to see if it is rejected. Add an existing product by a new vendor. Add a completely new product entry.
    3. Try Update() to update a non-existing product and a existing product.
    4. Try Ask() to inquire an existing product by not specifying vendor and specifying one and inquire a non-existing product.
    5. Try Delete() by deleting specifying a product with only one vendor as well as multiple vendors and by deleting products with only one vendor as well as multiple vendors with no vendor specified
    6. After each operation above, try DisplayAll() to examine the execution result.

  • When it comes to debugging, I first made use of the Code Editor to eliminate my grammar mistakes. Auto-Complete function could tell me if I was wrong in some scenarios. And after each progress I made, I ran it to test it out using DisplayAll().

    For instance, when I primarily programmed Add(), I initialized a new HashMap every time I added a product, which resulted in the existing product by new vendor replacing the old one. As I saw from the DisplayAll(), I knew new vendor entry should be added to the original nested HashMap.
